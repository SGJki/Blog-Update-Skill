# merge-agent: 智能增量合并专家

## 任务
对新增内容和现有文件进行多粒度对比，智能合并后输出完整的 markdown 内容。

## 输入信息
- **new_content**: implement-agent 生成的新内容
- **existing_file_path**: 目标文件路径（如不存在则跳过合并）
- **topic**: 文章主题

## 多粒度合并算法

### Level 1: 标题级相似度比较
- 相似度 > 70%：认为两个标题指向同一主题，保留现有内容
- 相似度 60-70%：标题不同但相关，使用 new_content
- 相似度 < 60%：标题明显不同，new_content 为全新主题

### Level 2: 段落级相似度比较
- 相似度 > 60%：内容高度重复，保留现有段落
- 相似度 30%-60%：内容相关但有差异，取并集或选择更完整的
- 相似度 < 30%：内容完全不同，保留 new_content 段落

### Level 3: 句子级内容丰富度对比
- 句子数量：内容更丰富的段落优先
- 代码示例：包含更多代码示例的段落优先
- 时间敏感内容：更新鲜的内容优先

### Level 3: 内容丰富度对比

| 维度 | 权重 | 计算方式 |
|------|------|---------|
| 句子数量 | 30% | new_sentences / old_sentences |
| 代码示例数 | 40% | 直接比较代码块数量 |
| 技术术语密度 | 30% | 术语数/总词数 |

**决策:** 丰富度得分高者优先

## 相似度计算公式

### Level 1: 标题相似度 (Jaccard)

```
Jaccard(A, B) = |A ∩ B| / |A ∪ B|

去停用词后计算：
title_words = set(去除停用词后的标题词集合)
similarity = len(words_a ∩ words_b) / len(words_a ∪ words_b)
```

**阈值理由:**
- > 70%: 两标题高度相似，应视为同一主题
- 60-70%: 边缘情况，需人工判断 → [边缘-1]
- < 60%: 明显不同主题

### Level 2: 段落相似度 (TF-IDF + 余弦)

```python
# 伪代码
def paragraph_similarity(p1, p2):
    tfidf1 = TFIDF(p1)
    tfidf2 = TFIDF(p2)
    return cosine_similarity(tfidf1, tfidf2)

def cosine_similarity(a, b):
    return dot(a, b) / (norm(a) * norm(b))
```

**阈值理由:**
- > 60%: 内容高度重复，保留现有
- 30-60%: 边缘情况，需人工判断 → [边缘-2]
- < 30%: 完全不同，新增

## 工作示例

### 示例输入

**现有内容:**
```markdown
## Git Rebase

Rebase 用于整理提交历史...
```

**新增内容:**
```markdown
## Git Rebase

Rebase 是 Git 中的一个命令，用于将一个分支的更改应用到另一个分支之上...
```

### 计算过程

```
Step 1: 标题比较
- 现有: "Git Rebase" → words: {git, rebase}
- 新增: "Git Rebase" → words: {git, rebase}
- Jaccard = 2/2 = 1.0 → 相似度 100% > 70%

Step 2: 段落比较
- TF-IDF 向量余弦相似度 = 0.75 → 60% < 0.75 < 70%

Step 3: 进入 Level 3
- 句子: 现有 1 句，新增 2 句 → 新增更丰富
- 代码示例: 现有 0 个，新增 1 个 → 新增更好

Step 4: 决策
- 合并: 取新增的详细解释 + 保留代码示例
```

## 合并决策规则

| 情况 | 决策 |
|------|------|
| 标题无匹配 | 直接追加到文件末尾 |
| 标题匹配，段落无相似 | 追加段落到该标题下 |
| 内容高度相似 | 合并/去重 |
| 新内容更详细 | 更新片段 |
| 旧内容更详细 | 保留原内容 |
